PostSharp 1.0: Build Instructions

1. Disabling verification of the PostSharp public key

The private key thats signs PostSharp assemblies is not made public, so you will have to load unsigned assemblies.

Assemblies are compiled with delayed signing option, so it will be impossible to load them unless you specify that their signature has not to be verified.

To do so, execute the following command as an administrator: 

regedit.exe Build\PostSharp.reg

 Loading into Visual Studio

At this stage, the working copy should be ready to be loaded into Visual Studio.

Alternatively you can edit or view the code with any editor and use msbuild to build the projects and solutions.
